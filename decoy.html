Decoy Mobile App

Laungauges and frameworks used

Decoy app is primarily a safe vault and file management system for any type of media file. A 7 in 1 app that acts as a media liabrary and player, sms messenger, photo viewer and file manager is designed to be one of the most secure apps for personal file storage and sms messaging. With up to 7 media catgories videos, images, audio, documents, sms, contacts, and a built in diary notebook decoy is a multitasking system that incoporates encryption technology through end to end encyption for sms messaging with AES and RSA sha 512 for all files stored providing high security and none breaching of personal files.

Decoy App as the name indicates is an app within another app, acts as two different apps in one through the possibility of switching modes, Public and private mode. Public mode is always the default mode whenever the app is launched at this mode decoy acts as a normal media libarary, sms messanger etc so anyone from the outside who opens your decoy app can only see your public files from your device internal storage that are not hidden, which gives the impression of a regular media liabrary and contact app. Private mode is enable by entering your password on any search bar of any media type category ,when the password is correct all public files disapear and decoy switches to private mode (a safe vault) with all hidden encrypted files and sms stored on your device throught decoy will be visible. For security reasons private mode operation state is only visbile as long as the user is currently using the app and terminates when the app is closed or when you manualy switch back to public mode, thus to prevent any outsider the ability to open your decoy app when its on private mode.


What Makes Decoy So Unique?


When private mode is enabled you have the ability to not only view encrypted hiiden files, but as well send encrypted sms messages as you would a normal sms. If you send a noraml sms with any sms messenger app including decoy in public mode it is sent in plain text and your service provider can read it and forwards it to your intended receipient, instead decoy private mode uses an end to end encryption protocol, the signal protocol to encrypt all sms passed through the wire. Your service provider wont be able to read the scrumbled encrypted text it recieves but only the intended receipient it forawrds it to can. Decoy has a special encrypted in built diary which allows a user to wirte and save stories,poems etc. Like many safe vault apps decoy is like a comaflage the empression of decoy is that is just a regular app that does nothing special until you enter your password activate private mode and the power of encrption comes to life.

encryption Features


  • Diary: hide and encrypt your personal stories
  • Video player: hide and encrypt your personal videos
  • Audio player: hide and encrypt your personal audio, music and recordings
  • Photo viewer: hide and encrypt your personal pictures
  • Document viewer: hide and encrypt your personal documents eg bank card info, social security number
  • Sms messenger: hide and encrypt your personal text messages as well send encrypted text messages down the wire so know one will be able to read it (including your service provider)
  • Contact list: hide and encrypt your personal contacts you will be able to hide all your call logs as well

for encrypting all types of files on your device, decoy uses a private and public key pair of RSA SHA 512 to encrypt all files. The private key is securely stored.

Sms encryption on the other hand works different as it uses the signal protocol with a AES 128 bit key. Your name is Alice and you want to send an encrypted sms to Bob You first have to create a secrete key out of band with Bob for it to work. Decoy is designed to be as decentralised as posiblie to desolve as much security valnabilities, to create a secrete with Bob you have to meet phyically with Bob or send each other you qr bar code images to scan (Although keys are signed for security reasons would not recommend), first create an encryption sms session with Bob tap on a key icon and let Bob scan your qr code which contains all your key certificates, let bob repeat the same and create a session as well allow you Alice to scan he's qr code to finalise the key exchange, go back to you qr bar code to see if both security numbers created for you and Bob match, if they macth you can now start sending encrypted sms. If the security numbers dont match then retry the process again until they match.
